Название проекта: "Приложение для перерисовки фотографий под одну из заранее определенных стилистик" (Visionary)
Отчётный период: 29.04.2025 – 06.05.2025
Команда: 6 человек (1 PM, 3 разработчика, 1 аналитик, 1 тестировщик)

1. Текущий статус проекта
Общий прогресс: 75% (проект в рамках графика).

Ключевые достижения за неделю:
— Разработаны экраны обработки изображений.
— Разработаны экраны постов (своего и чужого).
— Реализована авторизация/регистрация через сервер.
— Завершена загрузка моделей для обработки изображений.
— Создан черновой пайплайн обработки изображений.
— Настроены метрики.
— Разработан набор юнит-тестов.
— Подготовлен краткий отчет о тестировании.

2. Текущие задачи и проблемы
Задачи в работе:
— Реализация обработки изображений под стили через сервер.
— Реализация получения данных о профиле через сервер.
— Реализация получения изображений пользователя с сервера.
— Получение данных о посте через сервер.
— Завершение спринта в Jira.
— Начало нового спринта.
— Определение задач в бэклоге для третьего спринта и перенос их в состояние “к выполнению”.
— Обсуждение результатов предыдущего спринта и определение работы на предстоящую неделю.

Проблемы и риски:
- Загруженность фронтенд-разработчика. Сделан акцент на верстку всех экранов, что привело к неполной интеграции бэкенда и фронтенда, а именно к отсутствию обработки фото через сервер.(Решение: Задачи по интеграции будут выполнены на первой неделе 3 спринта вместо верстки экранов, которые фронтенд-разработчик сверстал заранее.)
- Невозможность продемонстрировать работу ИИ обработки изображений через сервер непосредственно с помощью интерфейсов.(Решение: Произвести обработку изображений с помощью привязанной нейросети и вставить готовые изображения в качестве мок-данных в готовый интерфейс.)

3. План на следующую неделю
Приоритеты:
— Реализовать обработку изображений под стили через сервер.
— Реализовать получение данных о профиле через сервер.
— Реализовать получение изображений пользователя с сервера.
— Реализовать получение данных о посте через сервер.

Метрики контроля:
— Реализована обработка изображений под стили через сервер к 12.05.
— Реализовано получение данных о посте через сервер к 12.05.
— Реализовано получение данных о профиле через сервер к 12.05.
— Реализовано получение изображений пользователя с сервера к 12.05.

4. Риски и их минимизация
Риск: Затянуть с интеграцией бэкенда и фронтенда.
Решение: Тщательный контроль за выполнением задач, напоминания от ПМ разработчикам, частые созвоны с отчетами о проделанной работе. Перестановка задач на временной шкале, чтобы не останавливать процесс разработки.

5. Комментарии команды
Команда сосредоточена на выполнении поставленных задач и активно работает над устранением выявленных ошибок. Разработчики готовы к расширенному взаимодействию для ускорения процесса разработки.

6. Заключение PM
Большая часть задач выполнена. Важно сосредоточиться на интеграции бэкенда и фронтенда в рамках третьего спринта. Использование мок-данных позволяет полностью продемонстрировать функциональность ИИ обработки изображений, даже если не будет полностью реализована интеграция через сервер. Также удастся продемонстрировать работу сервера и БД на реализованном процессе регистрации. Учитывая праздничные дни, команда показала очень хорошую продуктивность.
